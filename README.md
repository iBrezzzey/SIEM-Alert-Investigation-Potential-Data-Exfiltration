# SIEM-Alert-Investigation-Potential-Data-Exfiltration

## Objective

Assess the SIEM alert for potential data exfiltration, determine whether it is a true positive or false positive, and decide if escalation to a Level 2 (L2) analyst is required.

### Skills Learned

-SIEM Analysis: Experience analyzing alerts, logs, and correlation rules in SIEM tools.

-Incident Triage: Ability to distinguish true positives from false positives.

-Threat Detection: Identifying indicators of compromise, suspicious network traffic, and anomalous user behavior.

-Log Analysis: Parsing and interpreting system, application, and network logs for forensic investigation.

-Network Security Monitoring: Detecting potential data exfiltration, unauthorized access, or abnormal data transfers.

-Endpoint Awareness: Reviewing host activity to confirm legitimacy of alerts.

-Escalation Protocols: Understanding when to escalate to L2 analysts or higher-level incident response teams.

-Documentation & Reporting: Writing professional SOC tickets and investigation reports for compliance and auditing purposes.

### Tools Used

- Security Information and Event Management (SIEM) system for log ingestion and analysis.

Findings: Although the alert was initially triggered due to a large volume of data transfer, further analysis revealed that the traffic was directed to zoom.us, a legitimate video conferencing platform. The high network usage was generated by an active video call, which naturally produces significant data transfer. Therefore, the alert was not indicative of malicious activity or data exfiltration.

Conclusion: False Positive — no malicious activity detected.
